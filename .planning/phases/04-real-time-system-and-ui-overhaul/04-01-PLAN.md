Actually, examining the shadcn v4 + Tailwind v4 pattern: shadcn components in Tailwind v4 reference `bg-primary`, `text-primary-foreground` etc which map to `--color-primary`, `--color-primary-foreground` CSS variables through the `@theme inline` block.

    Our existing `--color-primary: #0F172A` IS the text color and components already use `text-primary`. This works for shadcn too since shadcn uses `text-foreground` for body text and `bg-primary` + `text-primary-foreground` for primary buttons. The collision is that our buttons use `bg-accent` but shadcn buttons use `bg-primary`.

    RESOLUTION per D-09 (shadcn provides structure, clinical system stays authoritative): Override shadcn button default variant in the generated `src/components/ui/button.tsx` to use `bg-accent text-accent-foreground hover:bg-accent-hover` instead of `bg-primary text-primary-foreground`. This is the correct approach — shadcn components are editable local files.
