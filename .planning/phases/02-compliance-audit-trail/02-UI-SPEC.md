## Component Inventory

### New Components

#### 1. ActivityTimeline

| Property | Value |
|----------|-------|
| File | `src/components/ActivityTimeline.tsx` |
| Props interface | `ActivityTimelineProps { entries: AuditLogEntry[] }` |
| Export | Named: `export function ActivityTimeline(...)` |
| Directive | `'use client'` |

**Layout:** Vertical timeline with left-aligned rail. Each entry is a horizontal row with:
- Left column (40px fixed): Nurse initial circle (28px diameter, `bg-surface border border-border`, centered letter in `text-xs font-semibold text-secondary`)
- Vertical connector line (1px `bg-border`) between entries, offset 14px from left edge of circle column
- Right column (flex-1): Action description, relative timestamp below

**Entry structure:**
```
[SC]  Sarah Chen approved flagged note
      2 minutes ago
```

**Timeline connector:** 1px solid `border` color, runs from bottom of one circle to top of next. First entry has no top connector. Last entry has no bottom connector.

**Sort order:** Newest first (`created_at DESC`), matching the note display convention.

**Accessibility:** `<ol>` semantic list. Each entry is an `<li>`. Timestamps use `<time datetime="...">`. Nurse initial circle uses `aria-label="{full nurse name}"`.

## Interaction Contracts

### I-01: Flag Badge State Progression (COMP-04)

**Trigger:** Nurse clicks action button in NurseActionBar
**Visual progression:**
1. Note starts with `warning` badge (amber, "Flagged")
2. Nurse clicks "Under Review" --> badge transitions to `under_review` (indigo, "Under Review")
3. Nurse clicks "Resolve" --> badge transitions to `resolved` (green, "Resolved")

**Transition animation:** None. Badge change is immediate on state update. Sonner toast confirms the action.

**State update pattern:** Optimistic UI -- badge updates immediately, reverts if Supabase UPDATE fails.

### I-02: Activity Tab Rendering (COMP-02, UI-09)

**Trigger:** Nurse clicks "Activity" tab
**Data fetch:** Query `audit_log` table filtered by `patient_id`, ordered by `created_at DESC`
**Loading state:** Skeleton pulse animation on 3 placeholder timeline entries (consistent with existing loading patterns)
**Populated state:** Chronological timeline with nurse initials, action descriptions, and relative timestamps
**Empty state:** Centered message (see Copywriting Contract below)

### I-03: Audit Entry Creation (COMP-01)

**Trigger:** Any of: approve, escalate, override, under_review, resolved (via trigger), confirm-supply, generate-handoff, create-note (via app-level insert)
**Visual feedback:** Sonner toast confirming the primary action (existing pattern). No separate toast for audit entry creation.
**Activity tab update:** If Activity tab is currently visible, new entry appears at the top of the timeline after re-fetch. No real-time subscription (deferred to Phase 4 RT-01).

### I-04: Terminal State Display (COMP-03)

**Trigger:** Note reaches a terminal status (approved, overridden, resolved)
**Visual change:** NurseActionBar buttons are replaced with a status line: "{Status} by {Nurse Name} at {time}"
**Style:** `text-sm text-secondary italic` with the status word in the corresponding semantic color (e.g., "Resolved" in `text-flag-safe`)
**Reversibility:** None. Terminal states cannot be undone.
