## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, `gap-1` |
| sm | 8px | Compact element spacing, `gap-2`, `py-2` |
| md | 16px | Default element spacing, `p-4`, `gap-4` |
| lg | 20px | Card internal padding, `px-5 py-4` (existing pattern) |
| xl | 24px | Section padding, `space-y-6` |
| 2xl | 32px | SOAP section vertical gaps (D-10 polish) |
| 3xl | 48px | Major section breaks between handoff cards |

Exceptions: Card internal padding uses 20px (`px-5`) throughout existing components. Phase 3 maintains this established pattern rather than forcing a shift to 16px or 24px.

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|-------------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.625 (`leading-relaxed`) | SOAP section content, summary paragraphs, list items, rationale text |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.5 | SOAP section headers (uppercase + tracking-wide), SOAP S/O/A/P labels (uppercase + tracking-widest), tab labels, timestamps, metadata, nested sub-section headers (HPI, Comorbidities, Interventions) |
| Summary | 16px (`text-base`) | 400 (regular) | 1.625 (`leading-relaxed`) | Handoff summary body text (visual prominence per D-14) |
| Heading | 18px (`text-lg`) | 600 (semibold) | 1.4 | Card titles: "Patient Context", "Supply Prep Recommendations", handoff section card titles |

**Weight discipline:** Phase 3 uses exactly 2 weights throughout:
- **400 (regular):** All body text, summary paragraphs, list items, rationale content, metadata values, timestamps
- **600 (semibold):** All labels, headings, SOAP section letters, nested sub-section headers, tab labels, card titles

SOAP S/O/A/P labels are differentiated from general section labels by `tracking-widest` (vs `tracking-wide`) rather than by weight. This follows the design system authority (`.interface-design/system.md`) which uses size and letter-spacing as the primary differentiators between label roles.

**Size discipline:** Phase 3 uses exactly 4 sizes:
- **12px** (`text-xs`): Labels, SOAP letters, timestamps, metadata, nested sub-section headers
- **14px** (`text-sm`): Body text, SOAP content, list items, rationale text
- **16px** (`text-base`): Handoff summary body (upgraded per D-14 for visual prominence)
- **18px** (`text-lg`): Card titles and section headings

Phase 3 typography changes (D-10, D-11):
- SOAP section labels: `font-semibold uppercase tracking-widest text-secondary` (upgraded tracking from `tracking-wide`)
- SOAP section vertical spacing: upgrade from `space-y-3` to `space-y-5` (12px to 20px gaps between S/O/A/P blocks)
- Nested sub-sections (HPI, Comorbidities, Interventions): use `text-xs font-semibold` (promoted from `text-[10px]` to 12px, differentiated from body by size and weight)
- Timestamp and metadata text: use `text-muted` consistently (already `#94A3B8`) for temporal markers to recede behind clinical content
- No line separators between SOAP sections -- white space only (Tufte 1+1=3 principle, per D-10)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--color-background`) | Page background, card internal backgrounds |
| Secondary (30%) | `#F8FAFC` (`--color-surface`) | Card containers, note cards, handoff report cards, context summary card |
| Accent (10%) | `#0EA5E9` (`--color-accent`) | Primary CTA buttons, procedure tags, "View Rationale" expand trigger, active tab indicator |
| Destructive | `#EF4444` (`--color-flag-critical`) | Critical flag badges in handoff report only |

Accent reserved for:
- "Generate Handoff Report" button
- "Mark All Ready" button on supply checklist
- Active tab indicator (Context tab, existing tabs)
- Procedure tags on notes
- "View Rationale" expand/collapse trigger text
- Supply link badges on note headers

Semantic flag colors (unchanged from existing system):
- `#F59E0B` (`--color-flag-warning`) + `#FFFBEB` bg: Warning flags in handoff priority flags
- `#EF4444` (`--color-flag-critical`) + `#FEF2F2` bg: Critical flags in handoff priority flags
- `#10B981` (`--color-flag-safe`) + `#F0FDF4` bg: Stable item indicators in handoff report
- `#6366F1` (`--color-flag-review`) + `#EEF2FF` bg: Under-review badge states (Phase 2)

No new color tokens required for Phase 3. All new UI surfaces use existing tokens.

#### 5. HandoffReport.tsx (D-12, D-13, D-14, D-15, UI-10)

Restructure from single card with border-b sections to dashboard card layout:

| Section | Card Spec |
|---------|-----------|
| Summary | `bg-surface border border-border rounded-lg p-5`. Heading at `text-lg font-semibold`. Body at `text-base leading-relaxed text-primary` (upgraded from `text-sm` for visual prominence per D-14). Top position. |

Layout: Vertical stack of 4 cards with `space-y-4` gap between them.

Visual style: lightweight borders (`border-border`), no shadows, desaturated (D-15).

## Interaction Contract

### Context Tab

| Interaction | Behavior |
|-------------|----------|
| Tab click | Switch to Context tab, display PatientContextSummary |
| Summary auto-update | After a new note is submitted, re-fetch patient summary (Supabase read, not a new Claude call) |
| No summary exists | Show empty state with guidance copy |

### Supply Rationale Expand/Collapse

| Interaction | Behavior |
|-------------|----------|
| Click "View Rationale" | Expand rationale block below header. Toggle text to "Hide Rationale". Chevron rotates 90 degrees (matches existing "Original Dictation" toggle in StructuredNote). |
| Click "Hide Rationale" | Collapse rationale block. Toggle text back. |
| Default state | Collapsed. Rationale hidden. |

### Handoff Report Cards

| Interaction | Behavior |
|-------------|----------|
| Report generation | "Generate Handoff Report" button triggers webhook POST. Button shows "Generating..." with `aria-busy="true"` during processing. |
| Report rendered | All four cards appear in vertical stack. No progressive reveal -- all sections render simultaneously. |
| No report exists | Show empty state with "Generate Handoff Report" CTA button. |
| Multiple reports | Show most recent report. Previous reports not displayed (consistent with current behavior). |

### SOAP Note Polish

| Interaction | Behavior |
|-------------|----------|
| No interaction changes | UI-04 is purely visual typography polish. No new click targets or state changes. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| SOAP section labels | Use `<dt>` / `<dd>` pattern inside a `<dl>` for SOAP sections (semantic improvement over current `<span>` + `<p>`) |
| Rationale toggle | `<button>` with `aria-expanded="true|false"` and `aria-controls="{rationale-id}"` |
| Rationale content | `<div id="{rationale-id}" role="region">` hidden/shown based on expanded state |
| Handoff flag alerts | Each priority flag card retains `role="alert"` (existing pattern) |
| Context summary | `role="status"` on summary container for screen reader announcement on update |
| Tab extension | New "Context" tab uses existing `role="tab"` / `aria-selected` / `aria-controls` pattern |
| Color contrast | All text meets WCAG 2.1 AA minimum: `text-primary` (#0F172A) on `bg-surface` (#F8FAFC) = 15.4:1 ratio; `text-muted` (#94A3B8) on white = 3.3:1 (used for non-essential metadata only, not content) |
